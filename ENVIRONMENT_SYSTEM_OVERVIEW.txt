                            USAGE EXAMPLES

DEVELOPMENT (DEFAULT)
  $ docker compose up --build
  → Uses .env.development values
  → Localhost endpoints, debug logging

CUSTOM PORT (QUICK TEST)
  $ API_PORT=5000 docker compose up
  → API runs on localhost:5000 instead of 4001
  → Other services on default ports

PRODUCTION DEPLOYMENT
  $ docker compose --env-file .env.production up --build
  → Uses production configuration
  → HTTPS enabled, info logging
  → Real credentials and endpoints (must be updated)

TESTING WITH VARIANTS
  $ MINIO_BUCKET=test-v2 QUEUE_NAME=test-queue docker compose up
  → Test with different storage bucket
  → Uses separate queue for testing
  → Other services on default values

MULTIPLE OVERRIDES
  $ NODE_ENV=staging API_PORT=5000 LOG_LEVEL=debug docker compose up
  → Multiple simultaneous overrides
  → No file changes needed
  → All variables work independently
